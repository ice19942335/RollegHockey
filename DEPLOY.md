# Инструкция по деплою на GitHub Pages

## Важно: Настройка base path

Перед деплоем **обязательно** обновите `base` в файле `vite.config.js` на название вашего репозитория на GitHub.

Например, если ваш репозиторий называется `my-hockey-tournament`, то:

```javascript
base: process.env.NODE_ENV === 'production' ? '/my-hockey-tournament/' : '/',
```

## Способ 1: Автоматический деплой через GitHub Actions (Рекомендуется)

1. **Включите GitHub Pages в настройках репозитория:**
   - Перейдите в Settings → Pages
   - В разделе "Source" выберите "GitHub Actions"

2. **Закоммитьте и запушьте код:**
   ```bash
   git add .
   git commit -m "Setup GitHub Pages deployment"
   git push origin main
   ```

3. **Проверьте деплой:**
   - Перейдите в раздел "Actions" вашего репозитория
   - Дождитесь завершения workflow "Deploy to GitHub Pages"
   - Ваш сайт будет доступен по адресу: `https://ваш-username.github.io/название-репозитория/`

## Способ 2: Ручной деплой через gh-pages

1. **Установите зависимости:**
   ```bash
   npm install
   ```

2. **Обновите base path в `vite.config.js`** (см. выше)

3. **Выполните деплой:**
   ```bash
   npm run deploy
   ```

4. **Настройте GitHub Pages:**
   - Перейдите в Settings → Pages
   - В разделе "Source" выберите ветку `gh-pages` и папку `/ (root)`
   - Сохраните изменения

## Проверка билда локально

Перед деплоем можно проверить билд локально:

```bash
npm run build
npm run preview
```

Откройте `http://localhost:4173` в браузере для проверки.

## Устранение проблем

### Проблема: Сайт не загружается или показывает 404

**Решение:** Проверьте, что `base` в `vite.config.js` соответствует названию вашего репозитория.

### Проблема: Ресурсы (CSS, JS) не загружаются

**Решение:** Убедитесь, что `base` путь заканчивается на `/` (например, `/Rolleg/`).

### Проблема: GitHub Actions не запускается

**Решение:** 
- Убедитесь, что файл `.github/workflows/deploy.yml` существует
- Проверьте, что вы пушите в ветку `main` (или измените название ветки в workflow)

## После деплоя

После успешного деплоя ваш сайт будет доступен по адресу:
```
https://ваш-username.github.io/название-репозитория/
```

**Важно:** При каждом изменении кода и push в ветку `main` будет происходить автоматический деплой (если используется GitHub Actions).
